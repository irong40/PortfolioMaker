# Audit Context — GIS Integration (2026-07-12)

Scope: the three commits `cbc2754..HEAD` on dev:

- `38b9073` feat: GIS integration — VARI veg analysis step, GIS exports, reel map card
- `c51ce22` refactor: dedup SRT track loading per function review
- `d376d1f` feat: GIS delivery policy per job type + delivery skips internal folders

## Intent

1. **gis_export.py** (new) — writes photo-points GeoJSON/CSV, per-clip flight-track GeoJSON, and a mission KML from photo EXIF GPS + DJI SRT telemetry. Runs on every job.
2. **vegetation_analysis.py** (new) — subprocess bridge to a headless QGIS VARI script (external, in drone-pipeline repo). Must degrade gracefully to None when QGIS is unconfigured or missing; must never crash the pipeline.
3. **portfolio_service.py** — pipeline wiring: GIS exports after manifest write; VARI step gated on preset flag `vegetation_analysis` + ortho existing. Delivery policy: presets with `gis_delivery: True` (property_survey, construction_progress, vegetation) put exports in `gis/` (delivered); all others in `_gis/` (internal). Client report Deliverables table must list only actually-delivered files. Same policy applies in both `process_job` and `portfolio_only`.
4. **drive_delivery.py** — new `collect_delivery_files()` walks output_dir skipping `_`-prefixed folders at any depth (also fixes a prior leak of `_report_thumbs/` and `_mipmap_work/` into client Drive packages).
5. **reel_render.py / reel_job.py** — 3.0s flight-path map card before outro on all packages; one polyline per clip SRT (never join separate flights); 0.5s telemetry decimation; timeline segmentation absorbs the card duration so reels still hit exact 45/60/90s targets. Card returns None gracefully when there is no SRT/KML data.

## Constraints

- Tkinter GUI monolith + pipeline layer; plain Python, no type checker enforced. Full suite: 427 tests pass (`python -m pytest`).
- GIS/VARI failures must NEVER fail a paying job — graceful skip everywhere.
- GeoJSON axis order is [lon, lat, alt]; SRT frames are ~30 Hz dicts with time_s/lat/lon/rel_alt.
- Out of scope: sortie_settings.json, rtklib/, .claude/ (pre-existing local state), and anything not touched by cbc2754..HEAD.

## Focus for This Audit

- Geo math correctness: lat/lon ordering, track decimation, equirectangular projection in the map card.
- Delivery-policy enforcement: can an internal `_gis/` file leak to a client package or client report under any path?
- Subprocess handling in vegetation_analysis.py: timeouts, malformed JSON output, missing files, path handling.
- Error-handling gaps that could crash a mission run.
